제목: AI로 만든 앱에 보안 문제가 12개 있는 줄 몰랐습니다

---

저는 개발자가 아닙니다. Cursor랑 Claude 써서 앱 만드는 사람이에요. 코드는 솔직히 잘 안 읽습니다. 돌아가면 배포합니다.

지난주에 레딧에서 누가 "AI가 짠 코드는 보안 문제가 많다"는 글을 올렸길래, "내 앱은 간단해서 괜찮겠지" 하고 넘겼는데 계속 신경 쓰여서 구글에 "내 코드 보안 검사" 검색해봤습니다.

나오는 도구들이 다 개발자용이더라고요. 뭘 설치하고 터미널에서 뭘 치고 YAML 어쩌고. 다 닫았습니다.

그러다가 GitHub 주소만 넣으면 검사해주는 사이트를 찾았습니다: https://vibesafe.onrender.com

제 레포 주소 넣었더니 30초 만에 결과가 나왔습니다.

**C등급. 12개 문제 발견.**

폼 input에 스크린리더가 읽을 수 없는 것들이 있다고 하고 (이거 때문에 소송당할 수 있다고?? 미국에서 2024년에만 접근성 소송이 4,000건 넘게 나왔답니다), 보안 패턴도 몇 개 나왔는데 설명이 이해할 수 있게 나와서 뭐가 문제인지는 알겠더라고요.

진짜 도움이 됐던 건 결과 맨 위에 "이걸 복사해서 AI에 붙여넣으세요"라는 박스가 있었습니다. 그거 복사해서 Cursor에 붙여넣었더니 Cursor가 다 고쳐줬습니다. 제가 뭐가 문제인지 이해할 필요도 없었어요. 복사 붙여넣기만 하면 됐습니다.

고친 다음에 다시 스캔했더니 100점 A등급.

누구나 해야 한다는 말은 아닙니다. 근데 저는 제 앱에 문제가 있는 줄 몰랐고, 이제 알아버렸으니 모른 척할 수가 없네요. AI로 뭔가 만들어서 실제 사용자한테 보여주려는 분이면 한번 URL 넣어보세요. 30초 걸리고 무료입니다.

참고로 이 사이트가 쓰는 스캔 엔진(Semgrep)은 Dropbox, Slack 같은 회사들이 쓰는 거고, 검사 기준(OWASP Top 10)은 보안 감사할 때 쓰는 국제 표준이라고 합니다. 아무 검사기가 아니라는 뜻.

아무튼 다시 만들러 갑니다.
