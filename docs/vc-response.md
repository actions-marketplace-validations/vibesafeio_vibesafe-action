## 문제 1 (비즈니스 모델) — 방정식

### 매출 공식
```
Revenue = Users × Conversion × ARPU × Retention
```

### 3가지 경로 분석

**경로 1 (B2C):** Revenue = 바이브코더 × 전환율 × $9/월 × 유지율
- 전환 동기: private repo 스캔 (진지해지면 private으로 전환)
- 문제: 바이브 코더는 위험을 모르므로 지불 의사 ≈ 0
- 판정: ❌ 단독으로는 성립 어려움

**경로 2 (B2B2C):** Revenue = 플랫폼 수 × 라이선스비
- Lovable/Bolt/Replit/v0/Cursor가 VibeSafe를 내장
- 각 $5K/월 → $25K/월 = $300K/년
- 그들의 핵심 역량은 코드 생성, 보안 검사가 아님
- 판정: ⚠️ 가능하지만 사용자 기반이 있어야 협상력 생김

**경로 3 (B2B):** Revenue = 팀 수 × 좌석 수 × $29/좌석/월
- AI 쓰는 주니어가 있는 팀의 시니어/리드가 도입
- 판정: ❌ 사용자 0명에서는 불가. 기반 구축 후 가능

### 결론
지금은 어떤 모델도 성립 안 함. 모든 경로의 전제 = 사용자 기반.
1주일 후 지표로 어떤 경로가 가능한지 판단.

## 문제 2 (Moat 없음) — 솔직한 분석

### 현재 moat: 없음
Semgrep 위에 YAML 8개 + 웹 UI. 주말에 복제 가능. 사실임.

### Moat 유형 MECE 분석
| 유형 | 적용 | 현실 |
|---|---|---|
| 네트워크 효과 | ❌ | 사용자끼리 상호작용 없음 |
| 전환 비용 | ❌ | YAML 한 줄 바꾸면 전환 |
| 규모의 경제 | ❌ | 서버 비용이 사용자에 비례 |
| 브랜드 | ⚠️ | 시간 필요 |
| 데이터 | ⚠️ | Semgrep이 더 많은 데이터 보유 |
| 유통 독점 | ✅ 유일한 기회 | 플랫폼 내장 시 채널 확보 |

### "왜 Semgrep이 직접 안 하나?"
Semgrep은 엔터프라이즈 ($40/dev/월)에 집중.
무료 웹 스캐너는 그들의 비즈니스 모델과 충돌.
**하지만 이건 그들이 전략을 바꾸면 사라지는 우위.**

### 솔직한 답
사용자 0명에서 moat 구축 불가.
유일한 전략: **속도** — 경쟁자보다 먼저 자리잡기.
Moat는 자리잡은 후 구축 (데이터 → 브랜드 → 유통 독점).
